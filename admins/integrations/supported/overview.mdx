This page lists every integration that ships with Aperium today, the auth model each one uses, and the fields an admin enters when configuring it.

<Note>
  Connector credentials are entered through the admin UI, either the **admin onboarding flow** the first time you sign in or the **Admin Console's MCP Servers tab** afterward, and are stored against the tenant. They are not configured through environment variables.
</Note>

## OAuth integrations

These connectors are linked per user from the Integrations page. The admin configures the OAuth client once (client ID, client secret, redirect URI); each user then consents individually from their own Integrations page.

| Connector | Identifier | Linked product surface | Admin-supplied OAuth client fields |
| --- | --- | --- | --- |
| Google Workspace | `google_workspace` | Gmail, Sheets, Docs | Client ID, client secret, redirect URI |
| Slack Workspace | `slack_workspace` | Channels, messages, files, search | Client ID, client secret, redirect URI |
| Atlassian | `atlassian` | Jira issues, Confluence pages | Client ID, client secret, redirect URI |
| Microsoft 365 | `microsoft365` | Outlook Mail, Calendar, OneDrive | Client ID, client secret, redirect URI |

Each connector has its own Link and Manage modal in the user-facing Integrations page, plus a scope selector that lets a user choose which capabilities to grant.

## Tenant-wide integrations with personal tokens

These connectors are set up once at the tenant level by an admin, and each user can add their own token, key, or username/password to use them. Some connectors require the personal token; others use the tenant-wide configuration as a default and let users optionally add their own.

### Required personal token

Each user must add their own credential before they can use the integration.

| Connector | Identifier | Description | Admin-supplied fields | Per-user fields |
| --- | --- | --- | --- | --- |
| Onyx | `onyx` | Onyx knowledge base search across Confluence, Drive, Slack, GitHub, Jira, and 40+ sources | Onyx URL | Personal Onyx token |

### Optional personal token

The tenant-wide credentials work for every user with a default permission set. Users can optionally add their own credentials if they want their own permission scope.

| Connector | Identifier | Description | Admin-supplied fields | Per-user fields (optional) |
| --- | --- | --- | --- | --- |
| Odoo | `odoo` | Odoo ERP integration | URL, database name, service user, service-user password | Personal service user and password |
| Salesforce | `salesforce` | Salesforce CRM integration | Instance URL, plus either a consumer key + consumer secret (client credentials flow) or username + password + token (password flow) | Personal username, password, and token |
| NetSuite | `netsuite` | NetSuite ERP integration | Account ID, consumer key, consumer secret, token ID, token secret | Personal token ID and token secret |

## Reviewing what is configured

To review which connectors are configured and active:

- Open the Admin Console and select **MCP Servers**. Each row shows the identifier, status, tool count, and edit/delete actions. System servers cannot be deleted.
- Or call `GET /api/mcp-servers` (admin-only) for the same data programmatically.
